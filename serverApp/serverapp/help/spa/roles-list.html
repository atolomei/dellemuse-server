Ayuda

Los roles sirven para otorgar permisos a los usuarios. Un usuario puede tener más de un rol asigando, y sus permisos son todos los que le otorgan los roles que tiene. Existen tres clases de roles, Generales, de Institución y de Sede:

  • Roles generales
    • Admin
      Es el rol de administrador general, tiene todos los permisos disponibles, puede ver y hacer todo menos editar el usuario de sistema ('root')
    • Audit
      Puede ver todos los contenidos del sistema pero no puede editar ni modificar nada
  • Roles de Institución
    • Admin
      Puede administrar la institución sus sedes. Ejemplos:
      Admin Institución( MALBA )
      Admin Institución( Museo Nacional de Bellas Artes )
  • Roles de Sede
    • Admin
      Puede gestionar y publicar los contenidos y administrar los usuarios de la sede. Ejemplos:
      Admin de Sede( MALBA )
      Admin de Sede( Museo Nacional de Bellas Artes )
    • Editor
      Puede editar todos los contenidos de la sede pero no puede gestionar la seguridad -editar usuarios, cambiar permisos, etc-, ejemplos: Editor de Sede( MALBA ), Editor de Sede( Cabildo de Buenos Aires )
